🧠 Mindset Before You Begin
Consistency > Intensity. Solving 2-3 problems daily beats 20 problems in one weekend.
Understanding > Memorization. Know why an approach works, not just what the approach is.
Struggle is learning. If every problem feels easy, you're not growing.
It's a marathon, not a sprint. Give yourself 3-6 months minimum.
Revision is non-negotiable. You will forget. Revisit solved problems regularly.


✅ Prerequisites
Before diving into DSA, make sure you're comfortable with:

One programming language (pick any — Python, Java, C++, JavaScript)
Basic syntax — loops, conditionals, functions
Basic math — modular arithmetic, prime numbers, powers of 2, logarithms
Understanding of memory — how variables, arrays, and references work
Time & Space complexity basics — Big O notation


🗺️ The Roadmap
Phase 1 — Build the Foundation (Weeks 1-3)
Topic	What to Learn
Time & Space Complexity	Big O, Big Omega, Big Theta, how to analyze loops
Mathematics	GCD, LCM, prime numbers, modular arithmetic, power functions
Recursion	Base case, recursive case, call stack, recursion tree
Bit Manipulation	AND, OR, XOR, left/right shift, checking/setting bits
🎯 Goal: Be able to analyze the time complexity of any code snippet and write basic recursive solutions.

Phase 2 — Core Data Structures (Weeks 4-8)
Topic	Key Concepts
Arrays	Traversal, insertion, deletion, rotation, prefix sums, sliding window
Strings	Palindromes, anagrams, pattern matching, string manipulation
Hashing	Hash maps, hash sets, frequency counting, collision handling
Linked Lists	Singly, doubly, circular, fast & slow pointers, reversal
Stacks	LIFO, monotonic stacks, expression evaluation, next greater element
Queues	FIFO, circular queue, deque, priority queue basics
🎯 Goal: Given a problem, be able to identify which data structure fits and implement operations from scratch.

Phase 3 — Algorithmic Techniques (Weeks 9-14)
Topic	Key Concepts
Sorting	Bubble, selection, insertion, merge sort, quick sort, counting sort
Binary Search	On arrays, on answer space, rotated arrays, search boundaries
Two Pointers	Pair sum, triplet sum, container with most water, merging
Sliding Window	Fixed size, variable size, longest/shortest subarray problems
Greedy	Activity selection, interval scheduling, Huffman-like thinking
Trees	Binary tree, BST, traversals (inorder, preorder, postorder, level order)
Heaps	Min heap, max heap, top-K problems, median finding
🎯 Goal: Recognize problem patterns and pick the right technique within minutes.

Phase 4 — Advanced Topics (Weeks 15-18)
Topic	Key Concepts
Graphs	BFS, DFS, adjacency list/matrix, connected components
Graph Algorithms	Dijkstra, Bellman-Ford, topological sort, union-find, Kruskal, Prim
Dynamic Programming	Memoization, tabulation, 1D DP, 2D DP, subsequences, knapsack
Backtracking	Permutations, combinations, N-Queens, Sudoku, subset generation
Tries	Insert, search, prefix matching, word dictionaries
Segment Trees / BIT	Range queries, point updates (if targeting competitive programming)
🎯 Goal: Solve medium-level graph and DP problems independently.

Phase 5 — Practice & Mastery (Ongoing)
Solve company-tagged problems on LeetCode
Participate in weekly contests (LeetCode, Codeforces, CodeChef)
Do mock interviews (Pramp, Interviewing.io, or with friends)
Revise using a spaced repetition system
Build a personal problem journal (see Tracking Progress)
🔍 How to Solve Any Problem
Follow this 5-step framework every single time:

text

Step 1 → READ the problem twice. Understand inputs, outputs, constraints, edge cases.

Step 2 → THINK of 2-3 examples manually. Draw diagrams if needed.

Step 3 → BRUTE FORCE first. What's the simplest (even if slow) approach?

Step 4 → OPTIMIZE. Can you reduce time complexity? Is there a pattern? 
         Apply known techniques (sorting, hashing, binary search, DP, etc.)

Step 5 → DRY RUN your solution on examples before considering it done.
⏱️ Time Rules While Practicing

💡 After looking at a solution: Don't just read it. Close it. Implement it yourself. Revisit it after 2-3 days.

❌ Common Mistakes to Avoid
Jumping to hard problems too early
→ Build intuition with easy/medium first.

Solving problems without understanding the pattern
→ Tag every problem with the technique used. Review patterns weekly.

Not revisiting solved problems
→ If you can't solve it again after a week, you didn't learn it.

Watching solutions without implementing
→ Always write the solution yourself after understanding.

Ignoring time/space complexity analysis
→ Every problem — analyze complexity. Make it a habit.

Skipping edge cases
→ Empty input, single element, all duplicates, negative numbers, overflow.

Studying too many resources at once
→ Pick ONE primary resource. Supplement only when stuck.

Comparing your pace with others
→ Everyone's starting point is different. Focus on your own growth.

The 3-Pass Technique for Tough Problems
text

Pass 1 → Attempt the problem completely on your own
Pass 2 → If stuck, read hints/approach (not full solution). Try again.
Pass 3 → If still stuck, study the solution. Understand every line. 
          Close it. Implement from memory. Revisit in 3 days.
